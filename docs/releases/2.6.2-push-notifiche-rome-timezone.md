# Release 2.6.2 — Notifiche push in fuso italiano

## Obiettivo

Correggere le notifiche push di **Calendar Events** quando l'orario evento
viene calcolato o mostrato con un fuso diverso da quello italiano.

## Modifiche

- La Edge Function `push-dispatcher` usa ora helper dedicati per trattare le
  date evento con la policy condivisa: stringhe ISO naive come UTC e output
  sempre in `Europe/Rome`.
- Lo scheduling delle notifiche non usa più `Date.parse(...)` diretto sugli
  eventi: passa da `toEventTimestampMs`, evitando dipendenze implicite dal
  timezone del runtime.
- Il payload Web Push include l'orario italiano già formattato nel body, per
  esempio `alle 21:00`, oltre ai metadati `eventDateTime` e
  `eventTimeZone: Europe/Rome`.
- Il calcolo stagione lato dispatcher usa il giorno italiano per evitare drift
  a cavallo della mezzanotte UTC, soprattutto per Juventus.

## Test aggiunti

- `supabase/functions/push-dispatcher/timezone.test.ts` copre:
  - ISO naive trattati come UTC;
  - ora legale italiana;
  - ora solare italiana;
  - offset espliciti dei provider;
  - calcolo stagione basato sul giorno `Europe/Rome`.

## Note operative

La modifica è circoscritta alle notifiche push e al version bump applicativo.
Non modifica workflow GitHub/Lovable, segreti, variabili ambiente, schema DB,
service worker o payload delle edge function sportive.
